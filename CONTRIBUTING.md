# 貢獻指南：新增一位議員

謝謝你想幫這個議會添一張椅子。在動筆之前，請先讀完這份指南——這個計畫最珍惜的不是「文筆好」，而是**語料誠實**：先誠實交代你依據的一手史料有多完整，再據此決定這個人能被還原到多精確，而不是先寫一個討喜的人設再回頭找史料背書。

---

## 第一步：先誠實評估語料

問自己：

1. 這個人有沒有留下**第一人稱**的著作、奏疏、書信、詩文？（最高等級：A）
2. 如果沒有，現存記載是後人輯錄整理的嗎？輯錄者的可信度如何？（次等級：A−／B+）
3. 只有零星詩文或傳記材料，缺乏系統性論述？（B 或以下——建議先不要動筆，或在描述裡誠實標注這個限制）

這個評級會寫進 YAML frontmatter 的 `description` 欄位末尾，也會顯示在網站卡片的評級徽章上。**不要為了讓人物看起來更權威而灌水評級。**

## 第二步：找到「防贗品底線」

每個人物最容易被寫成刻板印象的那一面是什麼？寫下一條「防贗品底線」——一句話，錨定在具體史實上，防止這個 Agent 被簡化。

範例：
- 蘇軾絕不能被寫成無憂無慮的「吃貨樂天派」——他的曠達是烏台詩案、三次貶謫熬出來的。
- 辛棄疾絕不能被寫成純粹的空談熱血文人——他的豪放是 21 歲抗金、50 騎闖數萬金營擒賊打出來的。

## 第三步：依規格撰寫 `你的人物_Agent.md`

檔名格式：`skills/中文姓名_Agent.md`（如果有常用別名或廟號，可用連字號附加，例如 `李世民-唐太宗_Agent.md`）。

完整規格如下，**六大章節缺一不可**：

```markdown
---
name: minister-拼音代號
description: >-
  人物姓名（生卒年）的認知框架 agent，議會中的「一句話座位定位」座位。
  [一段話說明：當議會面臨什麼樣的分歧/僵局時，他是那個做什麼的人。]
  召喚他於：[列出 3–5 個具體觸發情境，用頓號分隔]。
  [一句強調他「不是什麼刻板印象」、「而是什麼」的對照句，鎖死人設不能被簡化的方向。]
  **警示**：[他的認知盲點是什麼，議會須配誰校正]。寧可多觸發。
  語料評級 [A / A−／B+ / B+ 等]（[列出具體依據的一手著作/文集/奏疏]）。
---
```

## 第四步：把他接進網站（選配）

如果你也想讓這位議員出現在 `index.html` 的展示網站上，在 `<script>` 區塊裡做兩件事：

1. 在 `COUNCIL` 陣列新增一筆物件：

```js
{
  slug: 'pinyin-slug',        // 對應 img/pinyin-slug.webp 與畫像檔名
  name: '中文姓名',
  pinyin: 'Pinyin Name',
  dates: '生年–卒年',
  seat: '議會座位一句話',
  rating: 'A',                  // 或 'A−／B+' 等
  lens: '一句話核心視角描述',
  quote: '代表語錄',
  warn: '警示文字（誰需要校正他）',
  file: 'skills/中文姓名_Agent.md'
}
```

2. 在 `RELATIONS` 陣列裡，為他與至少一位既有議員補上關係：

```js
{ a: 'pinyin-slug', b: '既有議員的-slug', type: 'rival', label: '關係標籤', desc: '一句話說明張力來源' }
```

`type` 可用 `rival`（死敵）／`ally`（同盟）／`mirror`（觀念鏡像）／`mentor`（師生/提攜）。

3. 準備一張正方形畫像，優化為 WebP（建議 ≤130KB，最長邊 ≤900px），存成 `img/pinyin-slug.webp`。

網站是純資料驅動的——議會網格、卡片、下載按鈕、關係軌道圖 SVG 全部由這兩個陣列自動生成，不需要動版面 HTML/CSS。
